## 核心知识点速查

### 1. B+ 树

| 特点 | 说明 |
|------|------|
| 矮胖 | 磁盘 I/O 少 |
| 叶子链表 | 范围查询快 |
| 非叶子只存 key | 每页存更多，树更矮 |

### 2. 聚簇索引 vs 二级索引

| 索引类型 | 叶子节点存什么 | 是否回表 |
|---------|--------------|---------|
| 聚簇索引 | 整行数据 | 否 |
| 二级索引 | 主键值 | 是 |

### 3. 最左前缀法则

**核心**：查询条件必须从联合索引的最左列开始，不能跳过。

### 4. 索引失效清单（10种）
❌ LIKE '%xxx'（左模糊）
❌ 函数操作（LEFT、SUBSTR、DATE...）
❌ 隐式类型转换（int vs varchar）
❌ 最左前缀不满足
❌ OR 条件
❌ != 或 <>
❌ IS NULL / IS NOT NULL（可能）
❌ 索引列计算
❌ NOT IN（可能）
❌ 范围查询后面的列

## 面试速答

| 问题 | 答案 |
|------|------|
| InnoDB 为什么用 B+ 树？ | 矮胖（I/O 少）；叶子链表（范围查询快） |
| 聚簇索引和二级索引的区别？ | 聚簇索引叶子存整行，二级索引叶子存主键 |
| 什么是回表？ | 二级索引查到主键后，回聚簇索引查整行 |
| 什么是覆盖索引？ | 索引包含查询所需的所有字段，不需要回表 |
| 最左前缀法则是什么？ | 联合索引必须从最左列开始，不能跳过 |
| 索引失效最常见的原因？ | 左模糊、函数、隐式转换、最左前缀 |
| 怎么查看 SQL 执行计划？ | 使用 `EXPLAIN` 命令 |
| type 哪个最好哪个最差？ | `const`/`ref` 最好，`ALL` 最差 |
