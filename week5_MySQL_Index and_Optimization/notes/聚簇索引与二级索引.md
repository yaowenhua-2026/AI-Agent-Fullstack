# 聚簇索引与二级索引

## 一、聚簇索引（Clustered Index）

**叶子节点存整行数据。**
[50] ← 非叶子节点：主键值
/
[30] [70]
/ \ /
[20] [40] [60] [80]
↓ ↓ ↓ ↓
┌────┐ ┌────┐ ┌────┐ ┌────┐
│id=20│ │id=40│ │id=60│ │id=80│
│name│ │name│ │name│ │name│
│age │ │age │ │age │ │age │
└────┘ └────┘ └────┘ └────┘

text

**特点**：
- InnoDB 必须有聚簇索引
- 主键就是聚簇索引
- 如果没有主键，InnoDB 会用第一个唯一索引作为聚簇索引
- 如果都没有，InnoDB 会生成一个隐藏的 row_id
- 叶子节点存**整行数据**

## 二、二级索引（Secondary Index）

**叶子节点存主键值，不是整行数据。**

```sql
CREATE TABLE user (
    id INT PRIMARY KEY,
    name VARCHAR(20),
    age INT,
    INDEX idx_name (name)      -- 二级索引
);
text
        [30]                    ← 非叶子节点：name 值
       /    \
    [20]     [45]
   /   \    /   \
[15]  [25] [40]  [50]
  ↓    ↓    ↓      ↓
 id1  id2  id3    id4          ← 叶子节点：主键值
特点：

可以有多个二级索引

叶子节点存主键值

需要回表才能拿到完整数据

三、回表（Table Lookup）
sql
SELECT * FROM user WHERE name = '张三';
执行过程：

text
二级索引                    聚簇索引
┌─────────┐               ┌─────────┐
│ name=张三 │ ──找到id──→ │ id=1    │
│   id=1   │               │ 整行数据 │
└─────────┘               └─────────┘
步骤：

走二级索引，找到 name='张三'，拿到 id

用 id 回聚簇索引查找

找到整行数据

回表 = 两次 B+ 树查找

四、覆盖索引（Covering Index）
索引包含了查询所需的所有字段，不需要回表。

sql
-- 创建联合索引（覆盖索引）
CREATE INDEX idx_name_age ON user (name, age);

-- 查询字段都在索引中，不需要回表
SELECT name, age FROM user WHERE name = '张三';
覆盖索引的好处：

减少回表

减少 I/O

提高查询性能

六、覆盖索引判断
查询	索引情况	是否回表
SELECT * FROM user WHERE name = '张三'	只有 idx_name	✅ 需要回表
SELECT name FROM user WHERE name = '张三'	只有 idx_name	❌ 不需要回表
SELECT id, name FROM user WHERE name = '张三'	只有 idx_name	❌ 不需要回表（id 在叶子节点）
SELECT name, age FROM user WHERE name = '张三'	只有 idx_name	✅ 需要回表
SELECT name, age FROM user WHERE name = '张三'	有 idx_name_age	❌ 不需要回表
七、面试速答
问题	答案
聚簇索引叶子节点存什么？	整行数据
二级索引叶子节点存什么？	主键值
什么是回表？	二级索引查到主键后，回聚簇索引查整行数据
怎么避免回表？	用覆盖索引，把查询字段都放到索引里
一个表最多几个聚簇索引？	1 个
一个表最多几个二级索引？	多个
