# 联合索引最左前缀法则

## 一、联合索引是什么？

**联合索引 = 把多个列放在一起建索引**

```sql
CREATE INDEX idx_name_age ON user (name, age);
排序规则：先按 name 排序，name 相同再按 age 排序。

二、最左前缀法则（核心）
查询条件必须从索引的最左列开始，不能跳过。

联合索引	能走索引的条件
(a)	WHERE a = ?
(a, b)	WHERE a = ?（可有 b）
(a, b, c)	WHERE a = ?（可有 b、c）
核心：必须有最左列 a

三、能走索引 vs 不能走索引
联合索引 (name, age, sex)

WHERE 条件	能走索引吗	原因
name = '张三'	✅ 能	有最左列
name = '张三' AND age = 20	✅ 能	有最左列
name = '张三' AND age = 20 AND sex = '男'	✅ 能	有最左列
name = '张三' AND sex = '男'	🟡 部分能	有 name，但 age 被跳过，sex 用不上
age = 20	❌ 不能	没有 name
sex = '男'	❌ 不能	没有 name
age = 20 AND sex = '男'	❌ 不能	没有 name
四、Explain 验证
建表
sql
CREATE TABLE user (
    id INT PRIMARY KEY,
    name VARCHAR(20),
    age INT,
    INDEX idx_name_age (name, age)
);
验证结果
查询	type	key	说明
WHERE name = '张三'	ref	idx_name_age	✅ 走索引
WHERE name = '张三' AND age = 20	ref	idx_name_age	✅ 走索引
WHERE age = 20	ALL	NULL	❌ 不走索引（跳过 name）
WHERE age = 20 AND name = '张三'	ref	idx_name_age	✅ 优化器调整顺序
五、普通索引 vs 联合索引
维度	普通索引 (name)	联合索引 (name, age)
最左前缀	不需要	必须遵守
查询 WHERE age = 20	不走索引	不走索引（跳过 name）
查询 WHERE name = '张三'	走索引 ✅	走索引 ✅
覆盖索引	可能（只查 name）	更容易（name+age 都在索引里）
六、回表对比
普通索引
text
二级索引(name) → 找到 id → 回表查整行
联合索引（覆盖索引）
text
联合索引(name, age) → 叶子节点已有 name 和 age → 不需要回表 ✅
七、记忆口诀
最左前缀不能跳，从左到右依次找。

八、面试速答
问题	答案
联合索引 (name, age) 能走索引的条件？	WHERE 条件必须有 name
WHERE age = 20 AND name = '张三' 能走索引吗？	能，优化器会调整顺序
联合索引的好处？	多条件过滤、覆盖索引减少回表
为什么 WHERE age = 20 不走索引？	跳过了最左列 name
