# InnoDB 索引结构（B+ 树）

## 一、索引是什么？

**索引 = 书的目录**

- 没有索引：逐行扫描（全表扫描），慢
- 有索引：直接定位，快

## 二、InnoDB 为什么用 B+ 树？

### 各种数据结构对比

| 数据结构 | 特点 | 适合索引吗 |
|---------|------|-----------|
| 数组 | 插入删除慢 | ❌ |
| 链表 | 查找慢 | ❌ |
| 哈希表 | 不支持范围查询 | ❌ |
| 二叉树 | 可能退化成链表 | ❌ |
| AVL/红黑树 | 高度太高，磁盘 I/O 多 | 🟡 |
| B 树 | 每个节点存多个 key | ✅ |
| **B+ 树** | 叶子节点链表，范围查询快 | ✅✅ |

### B+ 树的核心优势

1. **矮胖** → 磁盘 I/O 少
2. **叶子节点链表** → 范围查询快
3. **非叶子只存 key** → 每页存更多，树更矮
