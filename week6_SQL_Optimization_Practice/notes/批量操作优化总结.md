# 批量操作优化

## 一、逐条插入 vs 批量插入

### 逐条插入

```sql
INSERT INTO user (name, age) VALUES ('user1', 1);
INSERT INTO user (name, age) VALUES ('user2', 2);
-- ... 重复 10000 次
批量插入
sql
INSERT INTO user (name, age) VALUES 
('user1', 1),
('user2', 2),
...
('user10000', 10000);
二、为什么逐条插入慢？
开销	说明
网络往返	每条 SQL 都是一次网络请求
SQL 解析	每条 SQL 都要解析、优化
事务开销	每条 SQL 单独事务
磁盘 I/O	每条 SQL 触发一次刷盘
三、性能对比（实验数据）
插入方式	耗时	网络往返	事务次数
逐条插入（1万条）	18.04 秒	10,000 次	10,000 次
批量插入（1万条）	0.28 秒	1 次	1 次
批量插入比逐条插入快约 65 倍！

四、优化技巧
1. 控制批量大小
批量大小	效果
太小（1-100）	网络开销仍大
合适（500-2000）	最佳
太大（>5000）	内存压力大，可能超时
2. JdbcTemplate 批量
java
jdbcTemplate.batchUpdate(sql, new BatchPreparedStatementSetter() {
    @Override
    public void setValues(PreparedStatement ps, int i) throws SQLException {
        ps.setString(1, list.get(i).getName());
        ps.setInt(2, list.get(i).getAge());
    }
    
    @Override
    public int getBatchSize() {
        return list.size();
    }
});
3. MyBatis 批量
xml
<insert id="batchInsert">
    INSERT INTO user (name, age) VALUES
    <foreach collection="list" item="user" separator=",">
        (#{user.name}, #{user.age})
    </foreach>
</insert>
4. 超大批量（百万级）
sql
LOAD DATA LOCAL INFILE '/tmp/data.csv'
INTO TABLE user
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n'
(name, age);
五、面试速答
问题	答案
批量插入为什么快？	减少网络往返、SQL 解析、事务开销
批量大小多少合适？	500-2000 条/批
百万级数据怎么插入？	LOAD DATA 或分批次批量插入
