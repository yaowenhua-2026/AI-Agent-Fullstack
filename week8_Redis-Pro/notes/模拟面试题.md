## 一、MySQL 面试题

### 1.1 索引相关

**Q1：为什么用 B+树？**

> 矮胖（磁盘 I/O 少）+ 叶子节点链表（范围查询快）

**Q2：聚簇索引和二级索引的区别？**

| 索引类型 | 叶子节点存储 | 是否回表 |
|----------|-------------|----------|
| 聚簇索引 | 整行数据 | 否 |
| 二级索引 | 主键值 | 是 |

**Q3：联合索引最左前缀法则？**

> 查询条件必须从最左列开始，不能跳过

**Q4：索引失效场景？**

| 场景 | 错误写法 | 正确写法 |
|------|----------|----------|
| 左模糊 | `LIKE '%张三'` | `LIKE '张三%'` |
| 函数 | `LEFT(name,2)='张三'` | `name LIKE '张三%'` |
| 隐式转换 | `phone = 13800138000` | `phone = '13800138000'` |

### 1.2 事务相关

**Q5：事务隔离级别？**

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|----------|------|-----------|------|
| READ UNCOMMITTED | ✅ | ✅ | ✅ |
| READ COMMITTED | ❌ | ✅ | ✅ |
| REPEATABLE READ（默认）| ❌ | ❌ | ✅ |
| SERIALIZABLE | ❌ | ❌ | ❌ |

### 1.3 优化相关

**Q6：慢 SQL 怎么优化？**

> 开启慢查询日志 → EXPLAIN 分析 → 加索引 / 优化 SQL

**Q7：大表分页怎么优化？**

-- 传统（慢）

-- 游标（快）

二、Redis 面试题
2.1 数据类型
Q8：五种数据类型及场景？

类型	适用场景
String	缓存、Session
Hash	用户信息、购物车
List	消息队列
Set	点赞、标签
Sorted Set	排行榜
2.2 过期与淘汰
Q9：过期策略？

惰性删除（访问时检查）+ 定期删除（每秒抽取一批）

Q10：淘汰策略？

allkeys-lru（最常用）、volatile-ttl（临时数据）、noeviction（不淘汰）

2.3 持久化
Q11：RDB vs AOF？

对比	RDB	AOF
原理	内存快照	追加写命令
恢复速度	快	慢
数据完整性	可能丢数据	最多丢1秒
2.4 缓存三大问题
Q12：缓存穿透、击穿、雪崩？

问题	解决方案
穿透	布隆过滤器、缓存空值
击穿	互斥锁、逻辑过期
雪崩	随机过期时间、缓存预热
2.5 分布式锁
Q13：Redisson 锁特点？

可重入、看门狗自动续期、公平锁、读写锁

Q14：看门狗机制？

锁默认 30 秒，每 10 秒检查，若仍持有则自动续期到 30 秒
