# 缓存与数据库一致性 - 学习总结

## 学习时间
2026-05-15

---

## 一、为什么需要缓存一致性？

在高并发系统中，Redis 作为缓存层，MySQL 作为持久层。当数据更新时，如果只更新数据库而不同步更新缓存，就会出现"脏读"：用户读到的是旧缓存数据。

**核心矛盾**：数据库和缓存的更新不是原子操作，高并发下易出现不一致窗口。

---

## 二、主流解决方案对比

### 2.1 方案对比表

| 方案 | 核心流程 | 优点 | 缺点 | 适用场景 |
|------|----------|------|------|----------|
| **Cache-Aside** | 先更新 DB，再删缓存 | 简单可靠 | 删除失败会脏数据 | **99% 场景** |
| **延迟双删** | 删→等→再删 | 一致性更高 | 降低写性能 | 对一致性要求稍高 |
| **双写+MQ** | 更新 DB→MQ→缓存 | 解耦 | 实时性降低 | 异步场景 |
| **Canal** | 监听 binlog→更新缓存 | 零侵入 | 引入额外组件 | 异构同步 |

---

## 三、Cache-Aside（最常用）

### 3.1 核心逻辑

```java
@Transactional
public void updateProduct(Product product) {
    productMapper.updateById(product);                    // 1. 更新DB
    redisTemplate.delete("product:" + product.getId());   // 2. 删除缓存
}
3.2 为什么删除而不是更新？
更新缓存代价高（可能涉及复杂计算）

删除后，下次读请求会重建缓存，简单可靠

四、延迟双删
java
@Transactional
public void updateProduct(Product product) {
    productMapper.updateById(product);
    redisTemplate.delete(key);
    Thread.sleep(500);  // 等待可能发生的并发读
    redisTemplate.delete(key);
}
原理：休眠时间 > 并发读请求从数据库读取 + 写入缓存的时间。

五、双写 + 消息队列
java
// 写请求
productMapper.updateById(product);           // 1. 更新数据库
mqProducer.send("update_cache", product);    // 2. 发送更新缓存消息

// MQ消费者
public void consume(Product product) {
    redisTemplate.opsForValue().set("product:" + product.getId(), JSON.toJSONString(product));
}
六、Canal 方案
6.1 Canal 是什么？
Canal 是阿里巴巴开源的数据同步系统，通过解析 MySQL 的 binlog，提供低延迟的数据订阅和消费功能。

6.2 工作原理
text
MySQL Master → Binlog → Canal(伪装成Slave) → Binlog解析 → 数据消费(更新缓存)
6.3 配置步骤
bash
# my.cnf 开启 binlog
[mysqld]
server-id=1
log_bin=mysql-bin
binlog-format=ROW
properties
# canal.properties
canal.destinations=example

# instance.properties
canal.instance.master.address=127.0.0.1:3306
canal.instance.filter.regex=.*\\..*
七、方案选型建议
场景	推荐方案
普通业务	Cache-Aside
对一致性要求稍高	延迟双删
需要解耦	双写+MQ
异构数据同步	Canal
八、面试速答
问题	答案
如何保证缓存一致性？	Cache-Aside：先更新 DB，再删缓存
为什么删除缓存而不是更新？	更新代价高，删除简单可靠
Canal 是什么？	监听 binlog，零侵入更新缓存
延迟双删解决什么问题？	解决并发读在删除前写入旧缓存的问题
