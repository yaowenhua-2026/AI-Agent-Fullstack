3.5 秒杀核心服务
java
public String seckill(Long productId, Long userId) {
    // 1. 防重复提交
    if (!checkUserToken(userId, productId)) {
        return "请勿重复提交";
    }
    
    // 2. 预检库存（锁外）
    String stockStr = redisTemplate.opsForValue().get("product:stock:" + productId);
    if (stockStr == null || Integer.parseInt(stockStr) <= 0) {
        return "已售罄";
    }
    
    // 3. 分段锁
    int segmentId = (int) (userId % SEGMENT_COUNT);
    RLock lock = segmentLocks[segmentId];
    
    try {
        boolean locked = lock.tryLock(5, 10, TimeUnit.SECONDS);
        if (!locked) return "系统繁忙";
        
        // 4. 原子扣减 Redis 库存
        Long newStock = redisTemplate.opsForValue().decrement("product:stock:" + productId);
        if (newStock < 0) {
            redisTemplate.opsForValue().increment("product:stock:" + productId);
            return "已售罄";
        }
        
        // 5. 发送 MQ 异步下单
        rocketMQTemplate.convertAndSend("seckill-order-topic", 
            new SeckillOrderMessage(productId, userId));
        
        return "抢购成功";
    } catch (InterruptedException e) {
        return "系统错误";
    } finally {
        if (lock.isHeldByCurrentThread()) {
            lock.unlock();
        }
    }
}

五、高并发优化总结
优化点	实现方式	效果
分段锁	userId % 16 分配锁段	并发度提高 16 倍
预检库存（锁外）	先读 Redis 判断库存	减少无效锁争抢
锁内只做扣减	只做 decrement 操作	锁持有时间降至几毫秒
防重复提交	setIfAbsent 令牌	防止恶意刷单
超卖回滚	<0 时 increment	保证库存不超卖
MQ 移到锁外	异步下单	网络 I/O 不占锁
