## 一、Redis 学习总结

### 1.1 五种数据类型及应用场景

| 类型 | 结构 | 特点 | 适用场景 |
|------|------|------|----------|
| **String** | 键值对 | 最简单，支持数字操作 | 缓存、Session、计数器 |
| **Hash** | field-value | 适合存对象 | 用户信息、购物车 |
| **List** | 双向链表 | 有序，可重复 | 消息队列、最新列表 |
| **Set** | 无序集合 | 无重复，支持交并差 | 点赞、标签、共同好友 |
| **Sorted Set** | 有序集合 | 按分数排序 | 排行榜、延迟队列 |

### 1.2 过期策略与内存淘汰

**过期策略**：
- 惰性删除：访问时检查过期
- 定期删除：定时抽取批量检查

**8 种淘汰策略**：

| 策略 | 范围 | 算法 | 适用场景 |
|------|------|------|----------|
| allkeys-lru | 所有 key | LRU | **通用缓存（最常用）** |
| volatile-lru | 有过期时间 | LRU | 混合场景 |
| allkeys-lfu | 所有 key | LFU | 防刷、统计 |
| volatile-ttl | 有过期时间 | TTL | 验证码、Session |
| noeviction | - | - | 不允许丢数据 |

### 1.3 持久化：RDB 与 AOF

| 对比项 | RDB | AOF |
|--------|-----|-----|
| 原理 | 内存快照 | 追加写命令 |
| 文件大小 | 小 | 大 |
| 恢复速度 | 快 | 慢 |
| 数据完整性 | 可能丢数据 | 最多丢1秒 |
| 适用场景 | 备份、快速恢复 | 数据安全要求高 |

**推荐配置**：RDB + AOF 混合持久化（Redis 4.0+）

### 1.4 主从复制与哨兵

| 组件 | 端口 | 作用 |
|------|------|------|
| 主库 | 6379 | 处理写请求 |
| 从库 | 6380、6381 | 读请求、数据备份 |
| 哨兵 | 26379、26380、26381 | 监控、自动故障转移 |

**高可用机制**：主库宕机 → 哨兵检测（5秒）→ 客观下线确认 → 选举新主 → 自动切换

### 1.5 事务与 Lua 脚本

**Lua 脚本优势**：
- 原子性：整个脚本一次性执行
- 支持条件判断
- 减少网络开销

**抢库存 Lua 脚本**：
```lua
local stock = redis.call('get', KEYS[1])
if stock and tonumber(stock) > 0 then
    redis.call('decr', KEYS[1])
    return 1
else
    return 0
end
