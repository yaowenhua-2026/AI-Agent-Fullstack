## 三、核心组件说明

### 3.1 自定义 Token

**文件**: `SmsCodeAuthenticationToken.java`

**作用**: 封装手机号和验证码，区分未认证/已认证状态。

| 状态 | principal | credentials | authenticated |
|------|-----------|-------------|---------------|
| 未认证 | 手机号 | 验证码 | false |
| 已认证 | UserDetails | null | true |

```java
// 未认证构造器
public SmsCodeAuthenticationToken(Object principal, String code) {
    super(null);
    this.principal = principal;  // 手机号
    this.code = code;            // 验证码
    setAuthenticated(false);
}

// 已认证构造器
public SmsCodeAuthenticationToken(Object principal, Collection<? extends GrantedAuthority> authorities) {
    super(authorities);
    this.principal = principal;  // UserDetails
    this.code = null;
    super.setAuthenticated(true);
}
3.2 自定义 Filter
文件: SmsCodeAuthenticationFilter.java

作用: 拦截 POST /login/sms 请求，提取手机号和验证码，创建未认证 Token。

java
public class SmsCodeAuthenticationFilter extends AbstractAuthenticationProcessingFilter {
    
    public SmsCodeAuthenticationFilter() {
        super(request -> "POST".equals(request.getMethod()) && "/login/sms".equals(request.getServletPath()));
    }

    @Override
    public Authentication attemptAuthentication(HttpServletRequest request, HttpServletResponse response) {
        String phone = request.getParameter("phone");
        String code = request.getParameter("code");
        SmsCodeAuthenticationToken authRequest = new SmsCodeAuthenticationToken(phone, code);
        return this.getAuthenticationManager().authenticate(authRequest);
    }
}
3.3 自定义 Provider
文件: SmsCodeAuthenticationProvider.java

作用: 校验验证码，调用 UserDetailsService 加载用户信息，认证成功返回已认证 Token。

java
@Component
public class SmsCodeAuthenticationProvider implements AuthenticationProvider {

    @Autowired
    private UserDetailsService userDetailsService;

    @Autowired
    private SmsCodeService smsCodeService;

    @Override
    public Authentication authenticate(Authentication authentication) {
        SmsCodeAuthenticationToken token = (SmsCodeAuthenticationToken) authentication;
        String phone = (String) token.getPrincipal();
        String code = (String) token.getCredentials();

        // 1. 校验验证码
        if (!smsCodeService.verifyCode(phone, code)) {
            throw new BadCredentialsException("验证码错误或已过期");
        }

        // 2. 加载用户信息
        UserDetails userDetails = userDetailsService.loadUserByUsername(phone);

        // 3. 返回已认证 Token
        return new SmsCodeAuthenticationToken(userDetails, userDetails.getAuthorities());
    }

    @Override
    public boolean supports(Class<?> authentication) {
        return SmsCodeAuthenticationToken.class.isAssignableFrom(authentication);
    }
}

3.5 UserDetailsService 扩展
文件: UserDetailsServiceImpl.java

作用: 根据用户名或手机号查询用户，支持两种登录方式。

java
@Service
public class UserDetailsServiceImpl implements UserDetailsService {

    @Autowired
    private UserMapper userMapper;

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        User user;
        // 简单判断：11位数字 → 手机号，否则 → 用户名
        if (username != null && username.matches("\\d{11}")) {
            user = userMapper.findByPhone(username);
        } else {
            user = userMapper.findByUsername(username);
        }
        if (user == null) {
            throw new UsernameNotFoundException("用户不存在");
        }
        return new CustomUserDetails(user);
    }
}
3.6 SecurityConfig 配置
java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity
public class SecurityConfig {

    @Autowired
    private SmsCodeAuthenticationProvider smsCodeAuthenticationProvider;

    @Bean
    public SmsCodeAuthenticationFilter smsCodeAuthenticationFilter(AuthenticationManager authenticationManager) {
        SmsCodeAuthenticationFilter filter = new SmsCodeAuthenticationFilter();
        filter.setAuthenticationManager(authenticationManager);
        // 设置成功/失败处理器
        return filter;
    }

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http,
                                           SmsCodeAuthenticationFilter smsCodeAuthenticationFilter) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/sms/**", "/h2-console/**", "/login/sms").permitAll()
                .anyRequest().authenticated()
            )
            .authenticationProvider(smsCodeAuthenticationProvider)
            .addFilterBefore(smsCodeAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
        return http.build();
    }
}
四、认证流程
text
1. 用户请求发送验证码
   GET /sms/code?phone=13800138000
   ↓
   SmsCodeController.sendCode() → 生成验证码，存储到内存

2. 用户提交登录
   POST /login/sms (phone=13800138000, code=123456)
   ↓
   SmsCodeAuthenticationFilter 拦截
   ↓
   创建 SmsCodeAuthenticationToken(未认证)
   ↓
   AuthenticationManager 调用 SmsCodeAuthenticationProvider
   ↓
   Provider 校验验证码 → 调用 UserDetailsService.loadUserByUsername(phone)
   ↓
   返回已认证 Token
   ↓
   SecurityContextHolder 存储认证信息
   ↓
   返回登录成功响应
